# Typeless Open UX Audit

日期: 2026-03-13

## 1. 音频抖动小窗口

### 1.1 Typeless 现象与机制判断

已确认的现象:
- Typeless 在全局唤起后会出现一个小浮窗。
- 用户说话时，浮窗会出现实时的音频抖动反馈。
- 不说话时，这个反馈会明显收敛或静止。

从本机 Typeless 包体中能确认到的信号:
- 浮窗是独立交互面，存在 `floating_bar` 文案和单独页面资源。
- 热键语义不是单一 toggle，而是多态交互:
  - 按一下开始/结束
  - 长按说一小段
  - 双击进入 hands-free 模式
- 有单独的 microphone 状态、设备切换、权限和错误提示文案。

基于这些信号的实现推断:
- Typeless 的浮窗不是“提交结果后才刷 UI”，而是录音期间就有实时状态机。
- 这个抖动效果大概率来自麦克风输入的实时电平分析，而不是 MediaRecorder 录完后回放。
- 常见实现会是 `getUserMedia + AudioContext + AnalyserNode`，按帧计算音量或波形，再驱动一个很轻量的可视化组件。

### 1.2 我们当前 App 是否具备类似能力

结论:
- 目前没有。

代码证据:
- 当前录音逻辑只做了 `MediaRecorder` 录音和时长计时。
- 没有 `AudioContext`、`AnalyserNode`、RMS/peak 计算或实时波形绘制。
- 浮窗当前只会显示:
  - 是否在录音
  - 录音时长
  - 是否已附加音频

对应代码:
- 历史实现曾位于旧 Electron 版本的 `apps/desktop/src/hooks/useVoiceRecorder.ts`
- 历史实现曾位于旧 Electron 版本的 `apps/desktop/src/components/FloatingPanel.tsx`

### 1.3 这项优化的明确目标

目标不是做“复杂波形编辑器”，而是做 Typeless 风格的即时反馈:
- 一开录音，浮窗立刻进入 active 状态。
- 说话时，浮窗中心或麦克风区域有节奏地跳动。
- 安静时，动画回落到低振幅。
- 不依赖云端结果，完全本地实时反馈。

### 1.4 建议实现顺序

1. 在 `useVoiceRecorder` 中新增实时音频分析链路
2. 输出 `inputLevel` / `smoothedLevel` / `isSpeaking`
3. 在浮窗中加入极简音频反馈组件
4. 再决定是否补“长按说话 / 双击免提”这种更进一步的交互

## 2. UI 界面布局优化

### 2.1 当前问题总结

当前首页虽然已经比最早版本更紧凑，但仍然偏“工具台”而不是“产品首页”:
- 信息入口仍然偏扁平
- 录音输入、结果输出、历史记录都在同一层竞争注意力
- 缺少“概览页”视角，无法像 Typeless 那样先看到自己的使用情况与设置
- 缺少统计、画像、反馈入口的分组

### 2.2 需要新增或重组的一级信息模块

建议把首页重构为 4 个一级区块:

1. 设置
- 快捷键
- 麦克风
- 输出语言
- 权限状态
- 账户 / provider 状态

2. 口述报告
- 总口述时间
- 总口述字数
- 节省时间
- 平均口述速度

3. 个人画像 / 大模型总结
- 用户长期使用画像
- 说话风格
- 常见上下文
- 建议优化点

4. 反馈与记录
- 最近转录
- 每次最终输出
- 失败重试
- 历史反馈 / 用户纠正入口

### 2.3 需要解决的结构问题

#### (a) 首页设置项过于扁平，需要重新组织

建议:
- 把设置从“散落按钮”改成一个单独 settings card / settings sheet
- 首页只保留最关键的设置摘要，不把所有控制都平铺在主界面

#### (b) 整合口述报告数据

建议:
- 首页出现一个 report summary 区
- 先做 4 个稳定指标:
  - 总口述时间
  - 总字数
  - 节省时间
  - 平均速度

数据来源建议:
- 先从本地 SQLite 历史记录推导
- 不依赖远程服务，保证 MVP 即可展示

#### (c) 优化大模型生成的个人报告展示

建议:
- 不要放成一大段说明文字
- 改成:
  - 一段摘要
  - 2 到 4 个标签
  - 2 到 3 个建议项

MVP 做法:
- 先基于历史记录生成本地 summary
- 没有真实 LLM 时，先用 mock summary 占位

#### (d) 重新设计反馈意见与转录记录的入口归类

建议:
- 把“即时输出”和“历史反馈”拆开
- 首页只展示最近几条记录
- 详细历史放到单独 History / Feedback 入口

## 3. 建议的优化顺序

### 第一优先级

音频抖动小窗口
- 这是 Typeless 体验里最容易被感知到的“活性”
- 也是当前我们和 Typeless 体感差距最大的地方之一

### 第二优先级

首页改成概览页
- 用“设置 + 口述报告 + 个人画像 + 反馈入口”重组信息架构

### 第三优先级

长按 / 双击 / 免提模式
- 这是更深入的热键交互层
- 价值高，但依赖前两步先把音频反馈和概览结构打稳

---

## 4. 当前结论

- Typeless 式音频抖动反馈，我们当前没有
- 这个功能应该优先通过本地实时音量分析补上
- 首页确实需要从“工作台”切到“概览页”
- 后续可以按下面顺序推进:
  1. 实时音频反馈
  2. 首页信息架构重做
  3. 长按 / 双击 / 免提模式
